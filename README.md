# WHRS人力资源管理系统

#### 介绍
人力资源管理系统（HRMS），主要功能包括员工主数据管理、入职到离职的全生命周期流程线上化、考勤数据对接、薪酬与社保计算，以及与OA和财务系统的集成。目标是实现人事管理的数字化，提升效率，降低错误，并确保数据准确合规。 

目标是构建统一、合规、高效的人事管理基础平台。

---

## 🎯 **总体目标**
1. 建立**唯一、准确、合规**的员工主数据源  
2. 实现**入职-在职-离职**全生命周期流程线上化  
3. 提供基础 **考勤功能** 和 **薪资计算功能** ，考勤打卡等数据源保留扩展能力，方便后期与OA工作流和财务系统对接
4. 替代 Excel/纸质管理，**降低事务性错误**

---

## 🧩 **五大核心功能模块**

### 一、组织与岗位管理
- **多层级部门架构维护**（集团→事业部→部门）
- **岗位与职级体系定义**（含汇报关系）
- **编制管理**（设置人数上限，超编预警）

---

### 二、员工主数据管理（系统核心资产）
- **标准化信息录入**（基础+工作信息，支持部分信息批量导入）
- **敏感信息加密存储【暂缓实现】**（身份证、银行卡等，符合《个人信息保护法》）
- **电子档案管理**（合同、证书等附件，可在线预览office和pdf等文件）
- **员工状态管理**（在职/试用/离职等）

---

### 三、员工全生命周期流程
- **入职管理**  
  - 劳动合同管理 + 入职信息清单  
  - 自动建档 
- **在职变更**  
  - 转正、调岗、晋升、入职、离职
- **离职管理**  
  - 离职
  - 自动冻结状态 

> 🔗 保留扩展可能性，本系统不提供工作流引擎，需要集成外部工作引擎

---

### 四、考勤数据对接（不自建打卡）
- **配置考勤规则**（班次、时间、规则）
- **本系统提供web打卡**（如需要支持设备打卡，钉钉、微信等需要扩展）
- **自动校验异常**（可通过扩展支持薪资计算）

> ⚠️ 明确策略：**只消费 OA 数据，不重复建设打卡功能**

---

### 五、薪酬与社保计算
- **灵活薪资结构配置**（支持多套方案）
- 可配置**自动社保公积金计算**
- 可配置**个税自动计算**
- **月度工资表生成**
- **电子工资条**

---


## ✅ **阶段成果价值**
- **数据统一**：打破信息孤岛，建立权威员工主数据  
- **流程提效**：人事事务线上化，减少人工干预  
- **合规保障**：满足个人信息保护、税务、社保等法规要求  


#### 软件架构
| 层级 | 技术选型 |
|------|--------|
| 前端 | Vue 3 (Composition API) + TypeScript + Vite + Element Plus + Pinia (状态管理) |
| 后端 | Spring Boot 3.4.x (Java 17) + MyBatis + MySQL |
| 部署模式 | 单体应用（Monolith），但内部高度模块化（Maven 多模块） |
| 安全 | Spring Security + JWT |

#### springBoot模块说明
| 功能需求 | 技术实现 |
|--------|--------|
| 员工主数据管理 | `farm2-service` + `farm2-base`（实体+DTO）+ MySQL |
| 敏感信息加密 | `farm2-tools`（AES/SM4 加解密工具） |
| 电子档案管理 | `farm2-files`（支持 PDF、Office、图片上传与预览） |
| 考勤对接  | `farm2-attendance` |
| 薪酬计算 | `farm2-salary`（JXLS 模板 + Aviator 表达式引擎） |
| 电子工资条 | `farm2-report`（生成 Excel/PDF） + `farm2-files`（存储/分发） |
| 全文检索 | `farm2-lucene`（员工档案、合同内容搜索） |
| 统一认证 | `farm2-auth`（JWT + Spring Security） |
